# OmniSite Tracker Launch Plan

## Phase 1: Testing & Polish

- [ ] Test on iPhone SE (small screen)
- [ ] Test on iPhone 15 Pro Max (large screen)
- [ ] Test data persistence across app restarts
- [ ] Test all placement sites log correctly
- [ ] Test edit/delete functionality in History
- [ ] Test recommendation algorithm with various data
- [ ] Fix any UI clipping or layout issues
- [ ] Verify Front/Back toggle works reliably
- [ ] Add haptic feedback on button taps
- [ ] Review accessibility (VoiceOver support)

## Phase 2: App Store Assets

- [x] Take screenshots (6.7" and 6.1" displays minimum)
- [x] Write App Store description (4000 char max)
- [x] Choose primary and secondary categories (Medical, Health & Fitness)
- [x] Select keywords (insulin pump, site rotation, diabetes tracker, etc.)
- [x] Create promotional text (170 char max)
- [ ] Record app preview video (optional)
- [x] Prepare "What's New" text for updates

## Phase 3: Legal & Compliance

- [x] Write Privacy Policy (required)
- [ ] Create Terms of Service (optional - not required for free apps)
- [x] Add medical disclaimer in app and store listing
- [x] Set up support email or URL
- [ ] Complete App Store age rating questionnaire
- [ ] Review Apple Health & Fitness app guidelines

## Phase 4: Technical Submission

- [ ] Enroll in Apple Developer Program ($99/year)
- [ ] Create App Store Connect app record
- [ ] Configure signing certificates and provisioning profiles
- [ ] Set bundle identifier and version number
- [ ] Archive build in Xcode
- [ ] Upload build to App Store Connect
- [ ] Submit for App Review

## Phase 5: Beta Testing

- [ ] Enable TestFlight in App Store Connect
- [ ] Invite 5-10 beta testers (T1D community)
- [ ] Collect feedback for 1-2 weeks
- [ ] Fix critical issues from beta feedback
- [ ] Upload final build

## Phase 6: Launch

- [ ] Set pricing (Free or Paid)
- [ ] Choose release option (manual or automatic after approval)
- [ ] Submit for final review
- [ ] Monitor review status
- [ ] Respond to any App Review rejections

## Phase 7: Post-Launch

- [ ] Monitor crash reports in Xcode Organizer
- [ ] Respond to App Store reviews
- [ ] Track downloads and usage
- [ ] Plan v1.1 with user-requested features
- [ ] Consider Apple Watch companion app

---

## Notes

**Estimated Timeline:** 2-4 weeks from testing to launch

**Key Risks:**
- App Review may require additional health disclaimers
- Need active Apple Developer account before submission

**Nice-to-Have (v1.1+):**
- iCloud sync across devices
- Notification reminders for site changes
- Export data to PDF/CSV
- HealthKit integration
